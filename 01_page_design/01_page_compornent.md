# サイト構成・記事詳細設計書

## 全体構造イメージ

* **Topページ（親）:** システムが自動集計した「最新状況まとめ」「ダッシュボード」。ユーザーはまずここを見て全体像を把握する。
* **Childページ（子）:** 特定のツイート（一次情報）を起点にした詳細記事。検索流入（SEO）の入り口となる。

```
🏠 TOPページ: ボンボンドロップ攻略ダッシュボード
（全情報の最新サマリ・各カテゴリへの入り口・最新アラート表示）

┃　　┗ 子: 【公式】1/10 サンリオコラボ入荷確定情報（公式ツイート＋詳細）
```

## 01. 📅 入荷予報・発売カレンダー

**役割:** 「いつ買える？」への回答。公式情報のハブ。

### 【Topページ】構成案

* **タイトル:** ボンボンドロップシール発売・再入荷スケジュール【随時更新】
* **メインコンテンツ:**
* **カレンダー表示:** 直近1ヶ月の「入荷予定日」「発売日」をリスト化。
* **ステータス信号機:**
* 🔴 **確定:** 公式発表あり (例: 1/10 サンリオ)
* 🟡 **噂・予測:** 過去傾向からの推測




* **自動リンク:** カレンダーの日付をクリックすると、該当する「Childページ」へ飛ぶ。

### 【Childページ】構成案 (記事フォーマット)

* **記事タイトル:** 【公式】1月10日、サンリオコラボ入荷確定！抽選販売の詳細まとめ
* **構成:**
1. **一次情報 (Tweet埋め込み):** 公式アカウントの告知ツイートを表示。
2. **要約・解説:**
* 「いつ？→1/10前後」
* 「どこで？→全国店舗＆オンライン」
* 「注意点は？→オンラインは抽選式」


3. **過去データからの予測:** 「前回のコラボ時は開店30分で整理券が終了しました。今回も朝イチ推奨です。」

## 02. 📍 みんなの目撃・完売ログ

**役割:** 「どこで売ってる？」への回答。地域・店舗SEO。

### 【Topページ】構成案

* **タイトル:** リアルタイム在庫・完売状況マップ【みんなの目撃情報】
* **メインコンテンツ:**
* **最新タイムライン:** 直近24時間の「あった！」「なかった…」情報を時系列で表示。
* **エリア別ステータス:** 「東京：品薄」「大阪：在庫あり報告多数」などをざっくり表示。


* **導線:** 「詳細を見る」ボタンで「Childページ」へ。

### 【Childページ】構成案 (記事フォーマット)

* **記事タイトル:** 【渋谷】エスターバニーのお店で大量目撃！12/30在庫速報
* **構成:**
1. **一次情報 (Tweet埋め込み):** ユーザーの「売ってた！」ツイートを表示。
2. **店舗情報解説:**
* 店舗名、地図（Google Map埋め込み）。
* 「ここは穴場です。ドンキが全滅でもファンシーショップには残っている傾向があります。」


3. **関連情報:** 「もしここも売り切れていたら、近くの〇〇もチェック。」

* **ひとこと分析:** 「今日はちいかわの完売報告が急増しています。」

3. **結論:** 「見つけたら即買い必須のレア度です。」

## 04. 📦 通販・オンライン在庫情報 (旧: 類似品図鑑)

**役割:** **【収益化の要】** 「ネットで買っちゃおう」への誘導。
**戦略:** X上の「これ買ったけど良かったよ（＋アフィリンク）」という文化を、自サイト内で再現・拡大する。

### 【Topページ】構成案

* **タイトル:** ネット在庫あり！今すぐ届く「ぷっくりシール」＆通販速報
* **メインコンテンツ:**
* **Amazon/楽天 在庫リスト:** システムが確認した「在庫あり」商品をタイル状に表示。
* **「みんなの戦利品」:** ユーザーがネットで買った代替品のレビューツイートを新着順に並べる。



### 【Childページ】構成案 (記事フォーマット)

* **記事タイトル:** 「ボンドロより安い！」Amazonで買えるウォーターシールが優秀すぎると話題
* **構成:**
1. **発端の口コミ (一次情報):**
* ユーザーの投稿：「娘っこ👧 ボンドロが一番だけど いっぱい欲しいから！って😂」という実際の声を引用。


2. **商品紹介・解説:**
* 「投稿で紹介されているのは、Amazonで販売中の『6枚セット ウォーターシール』です。」
* **比較:** 「本家は1枚400円ですが、これは6枚で1200円。コスパ最強です。」


3. **アフィリエイトリンク (CVポイント):**
* 大きなボタンや商品リンクカードを配置。
* 「Amazonで在庫を見る」「楽天で価格をチェック」


4. **クロージング:** 「『質より量』のお子様には、むしろこちらの方が喜ばれるかもしれません。」

### 💡 開発への影響・実装のポイント

この構成を実現するために、システム（Node.jsスクリプト）側で以下の処理が必要になります。

1. **カテゴリ自動分類機能:**
* スクレイピングしたツイートを、以下のルールでフォルダ分けして保存する。
* `official` → 01へ
* `location_name` あり → 02へ
* `sold_out_keyword` あり → 03の集計へ
* `amazon/rakuten_link` あり OR `recommend_keyword` ("代用", "似てる") あり → **04へ**
* `diy_keyword` ("自作", "レジン") あり → 05へ


2. **04番のアフィリエイトリンク変換:**
* ユーザーがツイートに貼っている短縮URL（`amzn.to/...`）を解析し、**あなたのアフィリエイトID（タグ）に置換する処理**、またはその商品の「あなたのリンク」を横に添える処理が必要です。
* *※他人のリンクをそのまま貼ると収益にならないため、商品特定→自リンク生成のフローが重要です。*
