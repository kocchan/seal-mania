
# 技術選定レポート：なぜ「シールマニア」にWordPressを採用するのか

## 1. 結論
**現段階（フェーズ1〜2）において、WordPressは「最適解」です。**

機能要件としては「Webアプリケーション（データベース管理・随時更新）」に近いですが、ビジネス要件（副業・1人運営・SEO集客）を考慮すると、フルスクラッチ開発（ReactやLaravel等）よりもWordPressの方が圧倒的に成功確率が高くなります。

---

## 2. Webアプリ開発 vs WordPress 比較評価

| 評価項目 | WordPress (CMS) | Webアプリ開発 (Next.js等) | シールマニアへの判定 |
| :--- | :--- | :--- | :--- |
| **SEO（検索集客）** | **◎ 最強**<br>初期状態で最適化済。記事を増やすほどドメインが強くなる。 | **△ 高難易度**<br>SSR等の高度な技術が必要。設定ミスでGoogleに認識されないリスクあり。 | SEOが生命線のため**WP** |
| **開発スピード** | **◎ 爆速**<br>1週間で公開可能。ブームを逃さない。 | **△ 遅い**<br>1〜2ヶ月かかる。その間にブームが去るリスクあり。 | トレンド重視のため**WP** |
| **維持・管理コスト** | **◎ 低コスト**<br>サーバー代のみ。機能追加もプラグインで数分。 | **△ 高コスト**<br>セキュリティ管理、バグ修正など技術的負債が増える。 | 副業・1人運営のため**WP** |
| **更新の操作性** | **◯ 工夫次第**<br>標準だとブログ向きだが、カスタマイズでExcel風に改造可能。 | **◎ 自由自在**<br>専用の管理画面を自作できるため最適化しやすい。 | 工夫で解決可能なため**WP** |

## 3. WordPressを「Webアプリ化」する構築戦略

標準の「ブログ機能」だけでは運用が回りません。以下のプラグイン構成により、**「裏側はデータベースアプリ、表側はSEOに強いメディア」**というハイブリッド構成を構築します。

### A. データベース構造の確立
記事（ブログ）とは別に、店舗データを管理する専用の「箱」を作ります。
* **CPT UI (Custom Post Type UI):**
    * 「投稿」とは別に**「店舗データ」**という投稿タイプを作成。
* **ACF (Advanced Custom Fields):**
    * 店舗データに「在庫状況」「確認日時」「エリア」などの項目（カラム）を持たせる。

### B. 更新オペレーションのアプリ化【重要】
「管理画面を開いて、記事編集画面に入って…」という手間を排除し、Excelライクな操作性を実現します。
* **Admin Columns (Pro推奨):**
    * 管理画面の記事一覧リスト上で、**直接「在庫状況」や「日時」を書き換えられる**ようにする。
    * これにより、移動中などにスマホから「数十店舗のステータス更新」が数分で完了します。

### C. 検索機能のアプリ化
* **Search & Filter Pro / VK Filter Search:**
    * 「埼玉県 × 在庫あり × イオン系列」のような**絞り込み検索機能**を実装。
    * ユーザー体験をWebアプリと同等にします。

## 4. 将来的な「卒業（移行）」のタイミング

WordPressで限界が来る（＝Webアプリ開発へ移行すべき）タイミングは明確です。ここまではWPで走り抜けることを推奨します。

1.  **リアルタイム同期が必要になった時**
    * 「ユーザーがボタンを押した瞬間に在庫数が減る」ようなEC機能が必要な場合。
2.  **複雑なユーザー投稿（CGM）機能**
    * マイページ機能、ポイント機能、ランク制度などを本格導入する場合。
3.  **月間100万PVを超えた時**
    * データベースの速度が限界に来た場合。
    * 👉 **この段階まで到達していれば、収益でエンジニアを外注してリニューアル可能です。**

## 5. 総括
今の「シールマニア」に必要なのは、高機能なプログラムではなく、**「いち早く情報を出し、Google検索で上位を取り、ユーザーに見つけてもらうこと」**です。

まずはWordPressで「見た目と機能はアプリ、中身はSEO最強のブログ」として立ち上げ、収益化を目指しましょう。
